```
Plan 1 (LLM Adapter) ──┐
                       ├─► Plan 3 (Direct Mode) ──► Plan 4 (Harness) ──► Plan 5 (Full v0.1)

```

- Plan 1 & Plan 2 can run in parallel (independent packages)
- Plan 3 needs both Plan 1 (LLM) and Plan 2 (Diff)
- Plan 4 needs Plan 3 (chat loop to integrate harness)
- Plan 5 needs Plan 4 (full chat+harness base for workflow UI)

## Execution Strategy

Each plan ends with independently testable software:

- **After Plan 1:** `awecode config` wizard works, `awecode chat-test` calls LLM
- **After Plan 2:** `@awecode/diff` package can parse + apply diffs, 100% unit tested
- **After Plan 3:** `awecode` (no args) opens Direct Mode TUI, can fix typos via chat
- **After Plan 4:** Agent can refactor + test in worktree with self-heal
- **After Plan 5:** Full success criterion — workflow engine, Repo Map, Compaction, 3-panel TUI

## Execution Options

Per `writing-plans` skill, each plan can be executed via:

1. **Subagent-Driven (recommended)** — fresh subagent per task, review between tasks
2. **Inline Execution** — batch execution with checkpoints in current session
